Key Invariant
Sessions never die. Executors can die. The harness can die. The network can partition. A session must always recover and accept the next message.
Fix Implemented
opencode proxy — dead-session recovery
When the harness returns 404 or is unreachable on a send path, the opencode proxy now automatically recovers the session without any user-visible error. The recovery sequence is:
  • Detect 404 / connection failure from downstream harness.
  • Create a new harness session against the (possibly replaced) pod.
  • Replay the full message history into the new session to restore context.
  • Retry the original send against the recovered session.
This change ensures the "sessions never die" invariant holds even when the underlying compute layer is killed, restarted, or rescheduled by Kubernetes.
Local Environment Notes
claude binary constraint + tests requiring full setup
The claude-agent-sdk harness requires the claude binary to be installed locally. It is not present on this machine. The harness handles a missing binary gracefully for single-session use (caught SDK errors are returned as valid responses), but crashes under concurrent load (10 sessions × binary ENOENT = unhandled process crash). Production on EKS has the binary and is not affected.
